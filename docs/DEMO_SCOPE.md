# 데모 범위 정의

## 데모의 목적

이번 데모의 목적은 최종 웹사이트를 완성하는 것이 아니라, 관련자들이 "우리 회사 홈페이지에 맞춘 CMS 관리도구"의 방향을 눈으로 확인하고 요구사항을 구체화하도록 돕는 것입니다.

따라서 데모는 다음을 보여주면 충분합니다.

- 정해진 디자인 안에서 텍스트와 링크를 바꾸는 흐름
- 뉴스/수상/제품 같은 반복 콘텐츠를 추가하거나 수정하는 흐름
- 섹션 순서를 바꾸면 공개 페이지에 반영되는 흐름
- 구독 영역처럼 한 섹션 내부 요소의 순서를 제한적으로 바꾸는 흐름

## 기준 디자인

데모 기준은 Figma `penta website demo` 캔버스와 `ref-image` 이미지 4개입니다.

- `C type_01` / `main_01.png`: 데이터 보안, D.AMO
- `C type_02` / `main_02.png`: 웹 보안, WAPPLES
- `C type_03` / `main_03.png`: 인증 보안, iSIGN
- `C type_04` / `main_04.png`: 클라우드 보안, Cloudbric

4개 화면은 대부분 같은 구조를 공유합니다. 데모 구현에서는 4개 페이지를 따로 만들지 않고, 하나의 메인 페이지 안에서 `ProductTabs` 상태만 바뀌는 구조로 구현합니다.

## 공개 페이지 구현 범위

### Header

편집 가능:

- 로고 대체 이미지 또는 텍스트 로고 표시 여부
- 상단 메뉴 텍스트
- 상단 메뉴 링크
- 검색 입력 placeholder
- 언어/유틸 아이콘 노출 여부

데모 제한:

- 실제 검색 기능은 구현하지 않습니다.
- 실제 다국어 라우팅은 구현하지 않습니다.

### Hero

편집 가능:

- 보조 문구: "믿을 수 있는 사회, 펜타시큐리티의 보안기술이 만들어 갑니다."
- 메인 문구: "Technology For Trust"
- 배경 도형/이미지 에셋

데모 제한:

- 도형의 자유 배치나 크기 조절은 제공하지 않습니다.
- 지정된 hero 프리셋 안에서 이미지만 교체하는 방식으로 표현합니다.

### News

편집 가능:

- 뉴스 제목
- 날짜
- NEW 배지 노출 여부
- "펜타시큐리티 소식 더 보기" 텍스트와 링크
- 뉴스 항목 추가/삭제/정렬

데모 제한:

- 뉴스 상세 페이지는 만들지 않습니다.
- 카테고리, 검색, 페이지네이션은 실제 요구사항 확정 후 판단합니다.

### Subscribe

편집 가능:

- 안내 문구
- 입력 필드 placeholder
- 버튼 텍스트
- 버튼 링크 또는 액션 타입
- `text`, `input`, `button` 요소 순서

권장 제약:

- 완전 자유 배치가 아니라 허용된 순서 프리셋 중심으로 구현합니다.
- 데모에서는 `text-input-button`, `input-button-text`, `text-button-input` 같은 프리셋을 보여줄 수 있습니다.

데모 제한:

- 실제 이메일 구독 저장이나 발송 시스템 연동은 구현하지 않습니다.
- 입력값 검증은 UI 수준의 간단한 예시만 둡니다.

### ProductTabs

편집 가능:

- 탭 라벨: 데이터 보안, 웹 보안, 인증 보안, 클라우드 보안
- 각 탭의 카테고리 문구
- 제품명 또는 로고
- 설명 문구
- 버튼 텍스트와 링크
- 제품별 시각 에셋

데모 데이터:

- D.AMO: 암호 플랫폼
- WAPPLES: 지능형 WAPP 솔루션
- iSIGN: 인증 보안 플랫폼
- Cloudbric: 클라우드 SaaS 보안 플랫폼

데모 제한:

- 제품 상세 페이지는 링크만 둡니다.
- 제품 에셋의 자유 편집은 제공하지 않고, 정해진 영역 안에서 교체만 허용합니다.

### Stats

편집 가능:

- 항목 라벨
- 숫자/문구
- 표시 순서

기본 항목:

- 고객 9000+
- 해외지사 일본 · 베트남 · 아랍에미리트
- 글로벌 고객 800+
- 글로벌 파트너 385
- 제품기술 수상 47
- 국내외 기술 특허 135
- 기술 인력 비중 60%+

### Awards

편집 가능:

- 수상/인증 로고
- 제목
- 설명
- 항목 순서

기본 항목:

- Frost & Sullivan
- Gartner
- Globee Award

데모 제한:

- 슬라이더 자동 재생이나 복잡한 캐러셀 기능은 1차 데모에서 제외합니다.

### Footer

편집 가능:

- Footer 메뉴 그룹명
- 그룹별 링크 텍스트와 URL
- 개인정보 처리방침 텍스트와 링크
- 회사 정보 문구
- copyright 문구

데모 제한:

- 실제 약관 페이지는 만들지 않습니다.

## 관리자 데모 범위

관리자 데모는 실제 Payload CMS가 아닙니다. 데모 미팅에서 편집 경험을 보여주는 임시 관리 화면입니다.

권장 화면:

- `/admin-demo`: 전체 편집 대시보드
- 섹션 순서 편집 영역
- Header/Footer 편집 영역
- News 편집 영역
- ProductTabs 편집 영역
- Subscribe 편집 영역
- Preview 영역 또는 공개 페이지 반영 안내

편집 방식:

- 텍스트 입력
- 링크 입력
- 반복 항목 추가/삭제
- 위/아래 이동 버튼 또는 드래그 정렬
- 이미지 필드는 데모에서는 파일명/프리셋 선택으로 처리 가능

## 데모에서 제외할 것

1차 데모에서 제외합니다.

- 실제 로그인/권한
- 실제 DB 저장
- 실제 MinIO 업로드
- 이메일 발송
- 뉴스 상세 페이지
- SEO 메타 관리
- 예약 발행
- 승인 워크플로우
- 감사 로그
- 완전 자유형 페이지 빌더

이 항목들은 데모 미팅 후 실제 요구사항으로 필요 여부를 판단합니다.

## 데모 성공 기준

- 마케터가 어떤 필드를 직접 수정할 수 있는지 이해한다.
- 기획자가 섹션 순서 변경과 자유 배치의 차이를 이해한다.
- 개발자가 실제 CMS 스키마로 옮길 수 있는 콘텐츠 구조를 확보한다.
- 관련자가 "데모에서 충분한 부분"과 "실제 프로젝트에서 추가해야 할 부분"을 구분할 수 있다.
